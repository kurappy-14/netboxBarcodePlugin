# NetBox Barcode Plugin テスト要件

## 1. 目的

本ドキュメントは、`netbox_barcode_plugin` の品質を担保するために必要なテスト要件を定義する。

本プラグインは、NetBox v4.6.1 Docker環境、Python 3.10以降、Django 5.0.x、Fetch API、Bootstrap 5、Material Design Icons、NetBox標準セッション認証を前提とする [1]。

テストでは、以下を重点的に確認する。

- プラグイン設定・URL・静的ファイルが正しく構成されていること
- バーコード照会APIが仕様通りに動作すること
- Cable検索、重複判定、権限判定が正しく行われること
- `trace()` 結果がA端/B端に分けて正しくJSON化されること
- `cable_status` 更新APIが権限・バリデーション・変更履歴記録を満たすこと
- フロントエンドがスマートフォン利用を想定したUI要件を満たすこと
- CDNに依存せず、ローカル配置された `html5-qrcode` を使用すること
- MIT License公開に必要なファイルが含まれていること

## 2. テスト対象

### 2.1 対象プラグイン

- Gitリポジトリ名: `netboxBarcodePlugin`
- Pythonパッケージ名: `netbox_barcode_plugin`
- Python配布パッケージ名: `netbox-barcode-plugin`
- PluginConfigクラス名: `NetBoxBarcodePluginConfig`
- NetBox表示名: `NetBox Barcode Plugin`
- base URL: `/plugins/barcode/`

| 項目 | 値 |
|---|---|

| value | label |

## 3. テスト方針

### 3.1 自動テスト

バックエンドおよびAPIは、原則として自動テスト対象とする。

推奨テスト方式:

- `pytest`
- `pytest-django`
- またはDjango標準の `TestCase`

テストコードは以下に配置することを推奨する。

```text
netbox_barcode_plugin/tests/
```

例:

```text
netbox_barcode_plugin/tests/
├── test_plugin_config.py
├── test_urls.py
├── test_lookup_api.py
├── test_permissions.py
├── test_status_update_api.py
├── test_trace_format.py
├── test_error_responses.py
├── test_static_assets.py
└── test_license_files.py
```

---

### 3.2 フロントエンドテスト

フロントエンドは、可能な範囲で自動テストし、カメラ・トーチなど端末依存が強い機能は手動テストも併用する。

推奨テスト方式:

- JavaScript単体テスト
- Playwright等によるブラウザテスト
- 手動実機テスト

カメラ制御、トーチ制御、HTTPS条件などはブラウザ・端末依存があるため、最低限の手動確認を実施する。

---

### 3.3 外部ネットワーク依存禁止

テストは外部CDNやインターネット接続に依存してはならない。

特に、`html5-qrcode` は以下のようなローカルstatic配下から読み込まれることを確認する。

```text
netbox_barcode_plugin/static/netbox_barcode_plugin/js/html5-qrcode.min.js
```

| 補足 | CSRFミドルウェア等のDjango標準エラーは別途扱いでもよいが、アプリケーション内で処理するAPIエラーはJSONで返すこと |

## 24. テスト対象外

以下は本テスト要件の対象外とする。

- NetBox本体の標準機能そのもののテスト
- `html5-qrcode` ライブラリ内部の単体テスト
- Docker環境への詳細な導入手順テスト
- 将来追加予定の `DEV-`、`RCK-` 等の具体的な検索機能
- ブラウザやOSごとのカメラAPI差異の完全な網羅

ただし、未対応プレフィックスが適切に拒否されること、および将来拡張しやすいプレフィックスディスパッチャー設計であることは確認対象とする。

## 25. 合格基準

リリース可能と判断するための最低条件は以下とする。

- P0テストがすべて成功していること
- バーコード照会APIの正常系・異常系が仕様通りであること
- ステータス更新APIで権限判定、値検証、Changelog記録が正しく動作すること
- 閲覧権限・更新権限がNetBox標準Object Permissionに従っていること
- 権限なしユーザーにCable詳細やtrace情報を返さないこと
- `html5-qrcode` がローカルstaticから読み込まれていること
- MIT Licenseおよび第三者ライセンス表記が含まれていること
- スマートフォンで最低限のスキャンまたは手入力運用が可能であること
