# NetBox Barcode Plugin インストール手順

---

## この手順でやること（全体像）

1. Docker で動く NetBox 環境（netbox-docker）を新規に用意する
2. このプラグインを `git clone` で取得する
3. プラグインを NetBox コンテナに読み込ませる
4. プラグインが必要とするカスタムフィールドを NetBox 側に作る
5. 画面用の静的ファイルを配信する
6. 動作確認する

所要時間の目安: 初回は Docker イメージのダウンロード込みで 15〜30 分ほどです。

---

## 前提条件

作業マシンに以下がインストールされている必要があります。

| 必要なもの | 確認コマンド | 補足 |
|---|---|---|
| Git | `git --version` | プラグインとnetbox-docker取得に使用 |
| Docker | `docker --version` | コンテナ実行基盤 |
| Docker Compose v2 | `docker compose version` | `docker-compose`（旧式）ではなく `docker compose` |

対応バージョンの目安:

- NetBox: **4.6 系**（このプラグインは `4.6.0`〜`4.6.99` に対応）
- Python / Django: NetBox 4.6 のコンテナに同梱のものを利用（自分で用意する必要はありません）

---

## ステップ 1: NetBox（Docker）環境を新規に用意する

公式の [netbox-docker](https://github.com/netbox-community/netbox-docker) を使います。作業用ディレクトリを決めて clone します。

```bash
# 作業用の親ディレクトリ（好きな場所でOK）
mkdir -p ~/netbox-barcode && cd ~/netbox-barcode

# netbox-docker を取得
git clone https://github.com/netbox-community/netbox-docker.git
cd netbox-docker
```

NetBox 4.6 系のイメージを使うため、`docker-compose.override.yml` を作成してバージョンとポートを固定します。
（このファイルは公式リポジトリには含まれないので、新規作成します）

`~/netbox-barcode/netbox-docker/docker-compose.override.yml` を以下の内容で作成します。

```yaml
services:
  netbox:
    image: netboxcommunity/netbox:v4.6-5.0.1
    ports:
      - "8000:8080"
  netbox-worker:
    image: netboxcommunity/netbox:v4.6-5.0.1
  netbox-housekeeping:
    image: netboxcommunity/netbox:v4.6-5.0.1
```

> ポイント
> - `image` のタグは NetBox 4.6 系を指定します（例では `v4.6-5.0.1`）。最新の 4.6 系タグは [Docker Hub の netboxcommunity/netbox](https://hub.docker.com/r/netboxcommunity/netbox/tags) で確認できます。
> - `8000:8080` は「PCの8000番ポート → コンテナの8080番」を意味します。ブラウザからは `http://localhost:8000` でアクセスします。

いったんこの時点で起動できるか確認します。

```bash
docker compose up -d
docker compose ps
```

`netbox` が `healthy` になれば OK です（初回はイメージDL＋DB初期化で数分かかります）。

---

## ステップ 2: プラグインを取得する（git clone）

NetBox 本体とは別の場所にこのプラグインを clone します。

```bash
cd ~/netbox-barcode
git clone https://github.com/kurappy-14/netboxBarcodePlugin.git
```

これで `~/netbox-barcode/netboxBarcodePlugin/` にソースが置かれます。
実際にプラグインとして読み込まれるのは、その中の `netbox_barcode_plugin/` ディレクトリです。

---

## ステップ 3: プラグインを NetBox に読み込ませる

プラグインを NetBox コンテナに認識させる方法は2通りあります。

- **方式A: bind mount（おすすめ・簡単）** … ホスト上のプラグインをコンテナにマウントするだけ。ソースを直せば即反映され、再ビルド不要。
- 方式B: pip install（配布向け） … イメージにインストールする。運用向けだが手間が増える。

ここでは **方式A** で進めます。

### 3-1. マウント設定を追記する

先ほど作った `docker-compose.override.yml` に、プラグインのマウントを追記します。最終的に次のようになります。

```yaml
services:
  netbox:
    image: netboxcommunity/netbox:v4.6-5.0.1
    ports:
      - "8000:8080"
    volumes:
      - /home/あなたのユーザー名/netbox-barcode/netboxBarcodePlugin/netbox_barcode_plugin:/opt/netbox/netbox/netbox_barcode_plugin:ro
  netbox-worker:
    image: netboxcommunity/netbox:v4.6-5.0.1
    volumes:
      - /home/あなたのユーザー名/netbox-barcode/netboxBarcodePlugin/netbox_barcode_plugin:/opt/netbox/netbox/netbox_barcode_plugin:ro
  netbox-housekeeping:
    image: netboxcommunity/netbox:v4.6-5.0.1
```

> 注意
> - パスは**絶対パス**で書きます。`~`（チルダ）は展開されないことがあるため、`/home/ユーザー名/...` のように書いてください。実パスは `cd ~/netbox-barcode/netboxBarcodePlugin && pwd` で確認できます。
> - 末尾の `:ro` は読み取り専用マウントの意味です。
> - `netbox` と `netbox-worker` の両方にマウントします（バックグラウンド処理側でも import されるため）。

### 3-2. プラグインを有効化する

netbox-docker では `configuration/plugins.py` にプラグイン名を書くと有効になります。

`~/netbox-barcode/netbox-docker/configuration/plugins.py` を開き、以下の内容にします。

```python
PLUGINS = ["netbox_barcode_plugin"]
PLUGINS_CONFIG = {}
```

既に他のプラグインを使っている場合は、`PLUGINS` のリストに `"netbox_barcode_plugin"` を追加してください。

### 3-3. 反映のためコンテナを作り直す

```bash
cd ~/netbox-barcode/netbox-docker
docker compose up -d
docker compose ps
```

`netbox` が `healthy` になるまで待ちます。

### 3-4. 読み込まれたか確認

```bash
docker compose exec netbox python /opt/netbox/netbox/manage.py shell -c \
  "from django.conf import settings; print(settings.PLUGINS)"
```

出力に `['netbox_barcode_plugin']` が含まれていれば成功です。

## ステップ 4: 必要なカスタムフィールドを作る

このプラグインは、NetBox の **DCIM > Cable** に以下2つのカスタムフィールドがあることを前提にしています。

| 名前 | 種類 | 役割 |
|---|---|---|
| `barcode` | テキスト | バーコードの値（例 `CBL-000001`） |
| `cable_status` | 選択（Selection） | ケーブルの状態（未作成 / 作成済み / 敷設済み） |

未設定のままだと、照会時に「カスタムフィールドが設定されていません」というエラーになります。

### 画面から作る場合（GUI）

1. まず管理者ユーザーが必要です。作成するには次を実行します。

   ```bash
   cd ~/netbox-barcode/netbox-docker
   docker compose exec netbox python /opt/netbox/netbox/manage.py createsuperuser
   ```

2. `http://localhost:8000/` にログインします。
3. **Customization > Custom Field Choices** で選択肢セットを1つ作成します。
   - 名前: `cableSelection`
   - 選択肢（値: 表示名）:
     - `not_created` : 未作成
     - `configured` : 作成済み
     - `laid` : 敷設済み
4. **Customization > Custom Fields** で2つ作成します。
   - `barcode`: Type = Text、対象オブジェクト = DCIM > Cable
   - `cable_status`: Type = Selection、選択肢セット = `cableSelection`、対象オブジェクト = DCIM > Cable

## ステップ 5: 画面の静的ファイルを配信する

プラグインの画面（スキャン・照会結果）は JavaScript / CSS を使います。これらを配信できるようにするため、`collectstatic` を実行します。

```bash
cd ~/netbox-barcode/netbox-docker
docker compose exec -u 0 netbox python /opt/netbox/netbox/manage.py collectstatic --no-input
```

> `-u 0` は root 実行の指定です。権限エラー（PermissionError）を避けるために付けています。

## ステップ 6: 動作確認

ブラウザで以下を開きます。

```text
http://localhost:8000/plugins/barcode/
```

- ログインしていない場合はログイン画面に飛ばされます（正常な動作です）。
- ログイン後、スキャン画面が表示されれば導入成功です。
- カメラが使えない環境（PCなど）では「手入力で検索」に、登録済み Cable のバーコード/ラベルを入れて照会できます。

> スマホのカメラでスキャンする場合、ブラウザのカメラ機能は原則 **HTTPS か localhost でのみ**動きます。`http://<PCのIP>:8000` のような HTTP 直アクセスではカメラが起動しないことがあります。その場合はリバースプロキシ等で HTTPS 化してください。手入力検索は HTTP でも使えます。

---

## うまくいかないときは

| 症状 | 主な原因 | 対処 |
|---|---|---|
| `/plugins/barcode/` が 404 | プラグインが読み込まれていない | `plugins.py` の記載と、ステップ3-4の確認をやり直す |
| 「カスタムフィールドが設定されていません」 | カスタムフィールド未作成 | ステップ4を実行する |
| 画面のデザインが崩れる / JSが動かない | 静的ファイル未配信 | ステップ5の `collectstatic` を実行する |
| ケーブルが「見つかりません」 | 一致する Cable が無い | `label` か `barcode` が完全一致（大文字小文字は無視）する Cable が必要 |
| 更新ボタンで権限エラー | 権限不足 | 対象 Cable への `dcim.change_cable` 権限、またはスーパーユーザーが必要 |

ログを見たいとき:

```bash
cd ~/netbox-barcode/netbox-docker
docker compose logs -n 200 netbox
```

## プラグインを更新したいとき

```bash
# 最新ソースを取得
cd ~/netbox-barcode/netboxBarcodePlugin
git pull

# 反映（Pythonコード変更時は再起動、JS/CSS変更時はcollectstatic）
cd ~/netbox-barcode/netbox-docker
docker compose restart netbox netbox-worker
docker compose exec -u 0 netbox python /opt/netbox/netbox/manage.py collectstatic --no-input
```

ブラウザは古い表示が残ることがあるので、ハードリロード（Windows: `Ctrl+Shift+R` / Mac: `Cmd+Shift+R`）してください。
